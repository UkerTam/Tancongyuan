项目背景

BACKGROUND

艾三维协同设计管理平台也被简称为CDMS系统，是一个同时管理结构化与非结构化信息的高效、开放、易于使用的协同工作平台。

企业通过此系统建立协同工作平台，从而在一个平台上实现了文档和信息的收集、加工、共享和发布到其他系统等功能。并有效提高项目管理的深度、提升协同工作的效率以及最大限度地保障企业内各种数字资产的安全。

用户群体

MAIN USER

主要用户为市政道路、建筑工程的设计院。设计院属于知识密集型企业，生成成品是电子文档，所以如果有效的管理控制设计成品，是设计院信息化要解决的核心问题。 经过统计设计人员30%的时间是用于设计相关资料的查找和收集，所以对于设计院来说，实现各种文档的共享和设计人员工作之间的协同设计，是提高设计效率的唯一手段。

赵先生

路桥工程师

由于图纸信息无法有效关联，我们设计时要找到相互关联的资料需要浪费大量时间自行查找、筛选。


协同设计中，经常出现参考错误的旧版图档而需重新设计的情况，影响了工作效率。

黄先生

设计院管理层

设计过程中产生大量重复性或相似性的电子图档，极易造成管理上的混乱，需要定制流程。


电子图档是设计单位的巨大财富，属于商业机密，如果缺乏相应的图档安全保密机制，档案很容易的被拷贝。

文档管理

DOCUMENT MANAGEMENT

文档管理页面所显示的是该用户当前所能查看的所有目录，可以在自己权限范围内查看所有文件。

文件上传功能需要文档目录拥有读写权限的用户才能上传，以避免恶意覆盖文件。

选择文件后可对此文件分别进行查看详细属性、预览文件、设置用户使用权限、选择用户对文件进行流程审核。

流程定义

PROCESS DEFINITION

拥有管理权限的用户可以自定义流程，并且可以定义每个状态变化时发生的动作。

流程状态变化时，可以发送消息、向个人工作台和向其他数据源写数据。

用户状态变化的时候，自动进行权限改变。

模型预览

MODEL PREVIEW

轻量化预览模型，在文档目录内放入展示用模型链接，可展示建筑模型。

能让户在此界面进行模型漫游，构建隔离，模型剖切等操作。
